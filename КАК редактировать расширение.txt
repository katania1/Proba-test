Режим разработки (Авто-сборка)
Когда ты садишься писать или менять код, открой терминал в папке extension и введи:

DOS

npm run build:dev
Что произойдет:

Rollup соберет код один раз, но не закроется.

Он перейдет в режим «слежки» (Watch Mode).

Теперь ты можешь открыть свои файлы в папке src/, писать код, и каждый раз, когда ты нажимаешь Ctrl + S (Сохранить), Rollup будет за миллисекунду сам автоматически пересобирать content.bundle.js в фоне! Терминал можно просто свернуть.

Важный нюанс про сам Chrome
Даже если Rollup мгновенно и автоматически обновил итоговый файл на жестком диске, Chrome агрессивно кэширует расширения. Поэтому алгоритм твоих действий при разработке такой:

Изменил код в src/какой-то-файл.js -> нажал Ctrl + S.

Перешел в браузер на страницу расширений (chrome://extensions/).

Нажал круглую кнопку «Обновить» (Refresh) на карточке VibeCoder.

Обновил саму вкладку с чатом Gemini (F5), чтобы инжектировался свежий скрипт.

А зачем тогда нужна была первая команда?
Команду npm run build (которую ты только что успешно выполнил) используют только один раз в самом конце, когда разработка фичи завершена. Эта команда применяет жесткую оптимизацию: минифицирует код, удаляет лишние пробелы и комментарии, делает файл максимально легким для финального «релиза». Режим build:dev этого не делает, чтобы собирать код мгновенно и оставлять его читаемым для отладки.
