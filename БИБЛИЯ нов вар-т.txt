
# 📘 VibeCoder — Ultimate Living Tech Spec (Project Bible)

## 1. Инструкция по ведению (Meta)

* **Единый источник истины:** Этот документ — единственное легитимное описание системы. Любое расхождение между кодом и Библией должно устраняться немедленным обновлением Библии.


* **Запрет на логи:** Хронологические записи о фазах разработки удаляются. Описывается только текущее состояние функций.


* **Мета-задача для ИИ:** При получении этого документа ИИ должен проанализировать структуру файлов (Раздел 3) и функционал (Раздел 4) для понимания контекста задачи.

## 2. Архитектурный Манифест (The Core)

**VibeCoder** — это локальная IDE и AI-оркестратор профессионального уровня для разработки через веб-версию Gemini и прямые API-запросы. Программа построена на принципах MVC и SOLID, изолируя UI от логики.

**Каналы транспорта данных:**

* **API-канал (Direct Provider):** Прямые запросы к OpenAI, Anthropic и др. через `APIWorker`. Данные передаются в виде полного мега-промпта (правила + проект + RAG + история) .


* **Чат-канал (Browser Bridge):** Работа через расширение Chrome. Использует Flask-сервер (порт 5070) для связи. Применяется «Гибридный Драг» и анти-фриз инъекции .

## 4. Топология интерфейса и функционал

### 4.1. Редактор кода (`editor.py`)

* 
**Визуализация:** Пунктирные направляющие отступов (`LookBoth`) и сплошные нити сворачивания (`BoxedTree`).


* 
**Управление:** Изолированная история Undo/Redo для каждой вкладки (`Alt + Left/Right`).


* 
**Навигация:** Плавающая кнопка прыжка (⬇ END / ⬆ TOP).


* 
**Безопасность:** Детектор внешних изменений файла и индикация несохраненного состояния (`*`).



### 4.2. Чат и Двухкнопочная система (`ui_main.py`, `chat_handler.py`)

* 
**[ 💬 Спросить ] (Enter):** Легкий режим чата без JSON и RAG .


* 
**[ ⚡ Кодить ] (Ctrl + Enter):** Тяжелый режим с JSON, RAG и Smart Diff .


* **Hybrid Drag v4.5:** Физические файлы (вложения, RAG-контекст) вешаются на курсор (`QMimeData`). Текст промпта идет через буфер обмена (`Ctrl+V`) .


* 
**Рендеринг:** Подсветка синтаксиса через `pygments` и кнопка "Копировать" с протоколом `copycode://`.



### 4.3. Файловый блок и Гит (`file_explorer.py`, `git_workflow.py`)

* **Explorer:** Тегирование файлов через `@[имя]`. Визуальное затухание путей из `.gitignore`.


* **Git:** Генерация AI-коммитов на основе `git diff`. Staging area через дерево чекбоксов.


* 
**Time Machine:** Откат версий из локальных `.bak` бэкапов через HTML-дифф.



### 4.4. Терминал и MCP инструменты (`terminal.py`, `mcp_manager.py`)

* 
**Терминал:** Эмулятор `cmd/bash` с перехватом Traceback для авто-отладки ИИ.


* 
**Инструменты:** Поиск DuckDuckGo, выполнение SQL и консольных команд . Агентный цикл ограничен 5 шагами.



## 5. Экосистема RAG и Контекст

* 
**RAG:** Использует ChromaDB с системой Auto-Healing (авто-восстановление при сбоях).


* 
**Индексация:** Жесткая изоляция `node_modules`, `.venv`, `dist` для экономии лимитов.


* **Биллинг:** Квоты сбрасываются в 10:00 утра. Ведется история расходов по каждому ключу.
