# 📘 VibeCoder v1.13 — Project Bible (Pro Edition)

## 1. Концепция проекта
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro.
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (горячие клавиши, зум, статистика токенов).

## 2. Технический стек
* **Язык:** Python 3.10+
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini.
* **База данных:** Локальные JSON-файлы (логирование истории чатов).

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**
* Профессиональная темная тема VS Code.
* Кастомный QTreeView (Дерево файлов) без лишних заголовков.
* Подсветка синтаксиса Python.
* Глобальные CSS-стили: стилизованные разделители панелей (QSplitter) и темные всплывающие подсказки (QToolTip).

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**
* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ.
* Ультимативный системный промпт: принудительный JSON, запрет заглушек.
* Автоматический Валидатор: если ИИ ошибается (ломает JSON или присылает ленивый код), скрипт САМ отправляет гневное сообщение с оригинальным промптом и заставляет ИИ переписать код (до 2-х попыток).

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**
* Обмен данными через `localhost:5070`.
* Единый контекст сессии: ИИ больше не сбрасывается после каждого ответа, диалог идет в одной ветке.
* Детектор лимитов: расширение читает уведомления Gemini и статус кнопки "Pro". Если лимиты исчерпаны, UI автоматически ставится на паузу и выдает кастомное темное предупреждение (`show_popup`).

**Фаза 4: Система Ревью (Diff Viewer) (Завершено ✅)**
* Live Preview: присланный ИИ код мгновенно отображается в редакторе (до сохранения).
* Модуль сравнения старого и нового кода (difflib) при нажатии "✅ Утвердить".
* Автоматическое создание бэкапов (.bak) перед любой перезаписью файла.

**Фаза 5: Логирование и Эстафета (Завершено ✅)**
* Сохранение каждой реплики в `chat_history.json`.
* Кликабельные ссылки в чате: эстафетный мега-промпт скрыт в логе, открывается по клику во всплывающем окне.
* Кнопка "🔄 Эстафета": формирует стартовый пакет, передает флаг `is_relay` в мост, после чего расширение само открывает новый чат Gemini.

**Фаза 6: Pro UX и Эргономика (Завершено ✅)**
* Единая нижняя панель управления (Dashboard) с иконками и динамической шириной кнопок.
* Независимый зум (Ctrl+Колесико) для трех зон: Ввод, Логи чата, Редактор кода. Сохранение настроек зума от сессии к сессии через системный реестр (`QSettings`).
* Живой математический счетчик токенов (Эвристика: ~2.5 символа = 1 токен) в нижнем Status Bar.
* Отправка запросов по `Ctrl + Enter` (обычный `Enter` работает как перенос строки).
* Умное масштабирование окна при запуске (адаптация под размер экрана).

**Фаза 7: Встроенный Файловый Менеджер (Завершено ✅)**
* Мини-панель над деревом проекта с кнопками "📄 Создать файл", "📁 Создать папку", "✏️ Переименовать", "🗑️ Удалить".
* Безопасное удаление с запросом подтверждения через системное диалоговое окно.

**Фаза 8: Навигация и Вкладки (Tabs) (Завершено ✅)**
* Внедрение `QTabWidget` над окном редактора кода.
* Отображение имени текущего активного файла (например, `📄 main.py`), что исключает путаницу.

**Фаза 9: Умное тегирование и Контекст (Drag & Drop + ПКМ) (Завершено ✅)**
* Возможность перетаскивать файлы из дерева прямо в чат (Drag&Drop).
* Автоматическая вставка тегов в формате `@filename.py` (или `@[file name.txt]` для имен с пробелами).
* Интеграция контекстного меню (ПКМ) по тегам в чате для опции «📎 Прикрепить код» (выделяется жирным зеленым) или открепления.
* Поддержка мульти-выделения (Shift/Ctrl) файлов в дереве с возможностью пакетной отправки их кода в чат через ПКМ.

**Фаза 10: Управление проектами и Security (Завершено ✅)**
* Кнопка `📂` для быстрого переключения между разными папками-проектами. Авто-сохранение последнего проекта.
* Защита от Path Traversal: программа блокирует попытки ИИ изменить файлы за пределами директории проекта.
* Добавлена опция просмотра "сырого" промпта по ссылке в чате.

**Фаза 11: Архитектурный рефакторинг (Завершено ✅)**
* Разделение монолитного `ui_main.py` на логические изолированные модули по принципу Single Responsibility.
* Выделение `custom_widgets.py` (умные текстовые поля, Drag&Drop, подсветка синтаксиса).
* Выделение `file_explorer.py` (управление деревом файлов, выбор проектов, контекстное меню).
* Значительное упрощение главного класса `MainWindow` для дальнейшего масштабирования IDE.

**Фаза 12: Система памяти (Smart Memory & Time Machine) (Завершено ✅)**
* Изолированная история проектов: папка `.vibecoder/` в корне каждого проекта для логов и бэкапов.
* Умная история сессий (History Dialog): разделение на список запросов и деталей (Master-Detail UI), быстрый текстовый поиск по вопросам и мыслям ИИ.
* Машина Времени: автоматическое сохранение 10 последних версий каждого файла при утверждении кода или ручном сохранении.
* UI отката версий: визуальное сравнение (Diff) старых бэкапов с текущим кодом и кнопка восстановления.

**Фаза 13: Умный патчинг и Авто-контекст (Smart Diff & Tool Calling) (Завершено ✅)**
* Smart Diff: ИИ больше не присылает полные файлы, а использует команды поиска и замены (`search`/`replace`), экономя токены и время.
* Конвейер Ревью: возможность ИИ изменять сразу несколько файлов. Программа открывает окна Diff поочередно для безопасного утверждения.
* Инструментальный вызов (Auto-context): ИИ может запрашивать файлы из дерева проекта, а программа (после подтверждения пользователя) автоматически извлекает их код и отправляет ИИ.
* Переход на математически точные Относительные Пути для всех файлов и тегов (нормализация слешей Windows/Unix).

## 6. Управление и Горячие клавиши
* `Ctrl + Enter`: Мгновенная отправка задачи из поля ввода.
* `Enter`: Перенос строки в поле ввода.
* `Ctrl + S`: Ручное быстрое сохранение открытого файла (с автоматическим созданием бэкапа в Машине Времени).
* `Ctrl + Колесико мыши`: Изменение размера шрифта (работает независимо в поле ввода, истории чата и редакторе кода). Все значения сохраняются автоматически.
* `Клик по ссылке [Показать текст] / [Показать сырой промпт]`: Просмотр скрытого системного промпта при смене сессии или перед отправкой.
* `Drag & Drop файлов`: Перетаскивание файлов из дерева в текстовое поле создает теги упоминаний с относительным путем (например, `@[core/main.py]`).
* `ПКМ по тегу @файл`: Прикрепляет или открепляет исходный код файла к отправляемому запросу (цвет тега меняется на зеленый).
* `ПКМ по файлу в дереве -> 🕒 Машина Времени`: Открывает интерфейс отката старых версий файла.
* `Shift/Ctrl + Клик в дереве`: Мульти-выделение файлов для пакетных операций.
