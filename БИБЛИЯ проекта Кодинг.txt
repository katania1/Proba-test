# 📘 VibeCoder v1.28 — Project Bible (Ultimate MVC + Smart RAG + Custom API)

## 1. Концепция проекта

**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM).
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (Semantic RAG, локальный Git, терминал, виртуальная файловая система).

## 2. Технический стек

* **Язык:** Python 3.10+
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)
* **Рендер чата:** Библиотеки `markdown` + `pygments` (HTML-генерация и подсветка синтаксиса)
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini, Virtual File System (Base64 -> React Setter / DragEvent), Высокоскоростная Fast-Track вставка и безопасный чанкинг через setTimeout(0) (Anti-Freeze для ProseMirror).
* **База данных и Память:** Локальные JSON-файлы (логи) + Локальная векторная БД ChromaDB (RAG с системой Auto-Healing)
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash).

* **Гибридная передача:** Легкие данные (Дерево проекта) идут текстом в промпте, тяжелые (RAG) — файлами.
* **Изоляция проектов:** Метод `RagController.cleanup()` принудительно гасит потоки и сбрасывает кэш ChromaDB при смене папки.
* **Anti-Thinking Selector:** Интеллектуальный выбор чистой модели "Pro/Advanced", избегая моделей с принудительным "Thinking", которые ломают JSON-ответы.
* **Умная ротация ключей (IndexerWorker):** Автоматическое переключение на резервные ключи из пула при исчерпании лимитов во время фоновой векторизации.

**Фаза 40: Бронебойная доставка кода и обход "Слепоты" Gemini (v1.28) (Завершено ✅)**
* **Отказ от VFS для файлов кода:** Исходный код из запросов `request_files` больше не упаковывается в скрытые Base64-вложения, так как внутренний парсер Gemini Web периодически их игнорирует ("Слепота ИИ"). Теперь запрашиваемый код вшивается напрямую в текстовый промпт, гарантируя 100% считывание моделью.
* **Оптимизированная вставка текста v6.2 Turbo Lightweight (content.js):** Полностью переработан механизм доставки промптов для предотвращения блокировки главного потока браузера (Main Thread Starvation) тяжелым текстовым движком ProseMirror.
  * **Безопасная очистка DOM:** Очистка поля ввода выполняется через Range API без использования деструктивной перезаписи `innerHTML` или опасной команды `execCommand('selectAll')`, что полностью исключает риск случайного синего выделения всей страницы.
  * **Щадящий Fast-Track:** Мгновенная вставка всего запроса через одиночный `execCommand('insertText')` применяется исключительно для коротких пакетов (длиной <= 4000 символов).
  * **Анти-фриз для ProseMirror (Safe Chunking):** Тяжелые запросы (> 4 КБ) гарантированно направляются в `TurboTextInjector` и вливаются безопасными блоками по 3500 символов через асинхронный `setTimeout(0)`. Это позволяет внутреннему `MutationObserver` редактора последовательно нормализовать DOM без зависания вкладки.
  * **Удаление мертвого кода:** Из каскада исключены заблокированные со стороны Gemini Web методы (React Fiber Injection и программный `ClipboardEvent` paste) для мгновенного старта без потерь времени на перебор путей.
  * **Агрессивные тайминги UI:** Предельное время слепого ожидания отрисовки UI-чипов загруженных файлов урезано с 15 до 3 секунд. Время ожидания активности (разблокировки) кнопки «Отправить» расширено до 5 секунд для компенсации лагов рендеринга.
* **Интеллектуальная очистка структуры проекта:** Модуль `ContextBuilder` теперь динамически парсит пользовательские списки `.gitignore` и `.vibeignore`. Мусорные папки (напр. `ВСЕ рабочие версии`, бэкапы) отсекаются еще на этапе `os.walk`, что исключает замусоривание памяти ИИ и экономит тысячи строк передаваемого дерева.

## 6. Управление и Горячие клавиши

* `Enter`: Отправка "Легкого" вопроса (💬 Спросить).
* `Ctrl + Enter`: Отправка "Тяжелой" задачи (⚡ Кодить).
* `Shift + Enter`: Перенос строки.
* `Ctrl + F`: Локальный поиск по коду.
* `Ctrl + S`: Ручное сохранение (+ локальный бэкап).
* `Ctrl + ~`: Показать/Скрыть терминал.
* `Ctrl + Колесико`: Независимый зум интерфейса.
* `Alt + Left / Alt + Right`: Undo / Redo действий в редакторе (работает для каждой открытой вкладки индивидуально).
* `Кнопка ⬇ END / ⬆ TOP`: Прыжок в конец или начало файла. Второй клик возвращает обратно.
* `Drag & Drop файлов`: Создание тегов `@[filename.py]`.
* `ПКМ по тегу @файл`: Прикрепить/открепить исходный код напрямую к промпту.
* `ПКМ по файлу в дереве -> 🕒 Машина Времени`: Визуальный откат из локальных бэкапов.
* `ЛКМ/ПКМ по слову "ВЫ"`: Открытие Инспектора или экспорт Debug Report.
