# 📘 VibeCoder v1.21 — Project Bible (Ultimate MVC + Smart RAG + Custom API)

## 1. Концепция проекта
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM).
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (Semantic RAG, локальный Git, терминал).

## 2. Технический стек
* **Язык:** Python 3.10+
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini
* **База данных и Память:** Локальные JSON-файлы (логи) + Локальная векторная БД (RAG)
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash).

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**
* Профессиональная темная тема VS Code.
* Кастомный QTreeView (Дерево файлов) без лишних заголовков.
* Подсветка синтаксиса Python.
* **Pro-Навигация (Стиль Notepad++):** Внедрены жесткие пунктирные направляющие отступов (`LookBoth`) и система сплошных светлых нитей сворачивания блоков (`BoxedTree`), обеспечивающие идеальную визуальную иерархию кода.
* Исправление конфликта окончаний строк (LF/CRLF), исключающее появление пустых строк при редактировании в сторонних IDE.

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**
* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ.
* Ультимативный системный промпт: принудительный JSON, запрет заглушек.
* Автоматический Валидатор: если ИИ ошибается (ломает JSON), скрипт САМ отправляет гневное сообщение и заставляет ИИ переписать код.
* Бронебойный экстрактор текста (Robust Parser) для сломанных JSON-ответов.

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**
* Обмен данными через `localhost:5070`.
* Единый контекст сессии (диалог идет в одной ветке).
* Детектор лимитов: расширение читает статус Gemini и автоматически ставит UI на паузу при перегрузке.

**Фаза 4: Система Ревью (Diff Viewer) (Завершено ✅)**
* Live Preview: присланный ИИ код мгновенно отображается в редакторе (до сохранения).
* Модуль сравнения старого и нового кода (difflib) при нажатии "✅ Утвердить".
* Автоматическое создание бэкапов (.bak) перед любой перезаписью файла.

**Фаза 5: Логирование и Эстафета (Завершено ✅)**

**Фаза 6: Pro UX и Эргономика (Завершено ✅)**

* Умная кодировка (CP866 -> UTF-8) и разделенный Split UX (логи отдельно от ввода).

**Фаза 20: Многопроектная архитектура (Portable Mode) (Завершено ✅)**
* Отказ от реестра Windows в пользу `.ini` (Portable Mode).
* Встроенный визуальный Менеджер Проектов.
* Авто-восстановление среды: умная детекция отсутствующего `venv` при клонировании с GitHub.

**Фаза 21: Векторная база знаний и Авто-индексация (Semantic RAG) (Завершено ✅)**
* Интеграция локальной векторной базы для семантического поиска по кодовой базе (Semantic Code Search).
* Фоновый QThread-обработчик для разбиения кода на чанки без заморозки интерфейса.
* Окно Аналитики RAG: отслеживание лимитов API (прогресс-бар), визуализация БД с точной математической сортировкой по числу чанков и дате.
* **Бронебойный Поллинг (Polling Engine):** Система автоматически (каждые 10 секунд) сканирует все файлы проекта. При сохранении файла в стороннем редакторе (VS Code, Notepad++), программа гарантированно обновляет RAG-индекс, полностью обходя баги "атомарного сохранения" операционных систем.

## 6. Управление и Горячие клавиши
* `Ctrl + Enter`: Мгновенная отправка задачи из поля ввода.
* `Enter`: Перенос строки в поле ввода.
* `Ctrl + S`: Ручное сохранение файла в редакторе (с созданием локального бэкапа).
* `Ctrl + ~`: Показать/Скрыть нижнюю панель терминала.
* `Стрелки Вверх/Вниз (в терминале)`: Навигация по истории команд.
* `Ctrl + Колесико мыши`: Изменение размера шрифта интерфейса.
* `Автоматическое RAG-обновление`: При сохранении файла в любом стороннем редакторе, IDE сама перестроит векторный индекс измененного файла в течение 10 секунд.
* `Клик по кнопке 📦 Git`: Открывает окно синхронизации и Staging Area.
* `Drag & Drop файлов`: Создание тегов упоминаний (`@[core/main.py]`).
* `ПКМ по тегу @файл`: Прикрепляет/открепляет исходный код файла (зеленый текст).
* `ПКМ по файлу в дереве -> 🕒 Машина Времени`: Откат файла из локальных бэкапов.
* `ПКМ по файлу в окне Git -> 🕒 Посмотреть Git-историю`: Откат файла до облачного коммита.
* `ПКМ по файлу в дереве -> 🚫 Добавить в .gitignore`: Прописать путь в игнор-лист.
* `Shift/Ctrl + Клик в дереве`: Мульти-выделение файлов.
* `ПКМ по модели в окне API:` Добавление текстового примечания к модели.
* `Клик по имени проекта (над деревом файлов):` Открывает Менеджер Проектов.
