

# 📘 VibeCoder v1.18 — Project Bible (Ultimate MVC Edition)

## 1. Концепция проекта
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro[cite: 4].
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (горячие клавиши, зум, статистика токенов)[cite: 4].

## 2. Технический стек
* **Язык:** Python 3.10+[cite: 4].
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)[cite: 4].
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)[cite: 4].
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)[cite: 4].
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini[cite: 4].
* **База данных:** Локальные JSON-файлы (логирование истории чатов)[cite: 4].

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**
* Профессиональная темная тема VS Code[cite: 4].
* Кастомный QTreeView (Дерево файлов) без лишних заголовков[cite: 4].
* Подсветка синтаксиса Python[cite: 4].
* Глобальные CSS-стили: стилизованные разделители панелей (QSplitter) и темные всплывающие подсказки (QToolTip)[cite: 4].

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**
* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ[cite: 4].
* Ультимативный системный промпт: принудительный JSON, запрет заглушек[cite: 4].
* Автоматический Валидатор: если ИИ ошибается (ломает JSON или присылает ленивый код), скрипт САМ отправляет гневное сообщение с оригинальным промптом и заставляет ИИ переписать код (до 2-х попыток)[cite: 4].
* Бронебойный экстрактор текста (Robust Parser): способность извлекать мысли ИИ и коммиты даже из сломанного JSON-ответа.

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**
* Обмен данными через `localhost:5070`[cite: 4].
* Единый контекст сессии: ИИ больше не сбрасывается после каждого ответа, диалог идет в одной ветке[cite: 4].
* Детектор лимитов: расширение читает уведомления Gemini и статус кнопки "Pro"[cite: 4]. Если лимиты исчерпаны, UI автоматически ставится на паузу и выдает кастомное темное предупреждение (`show_popup`)[cite: 4].

**Фаза 4: Система Ревью (Diff Viewer) (Завершено ✅)**
* Live Preview: присланный ИИ код мгновенно отображается в редакторе (до сохранения)[cite: 4].
* Модуль сравнения старого и нового кода (difflib) при нажатии "✅ Утвердить"[cite: 4].
* Автоматическое создание бэкапов (.bak) перед любой перезаписью файла[cite: 4].

**Фаза 5: Логирование и Эстафета (Завершено ✅)**
* Сохранение каждой реплики в `chat_history.json`[cite: 4].
* Кликабельные ссылки в чате: эстафетный мега-промпт скрыт в логе, открывается по клику во всплывающем окне[cite: 4].
* Кнопка "🔄 Эстафета": формирует стартовый пакет, передает флаг `is_relay` в мост, после чего расширение само открывает новый чат Gemini[cite: 4].

**Фаза 6: Pro UX и Эргономика (Завершено ✅)**
* Единая нижняя панель управления (Dashboard) с иконками и динамической шириной кнопок[cite: 4].
* Независимый зум (Ctrl+Колесико) для трех зон: Ввод, Логи чата, Редактор кода[cite: 4]. Сохранение настроек зума от сессии к сессии через системный реестр (`QSettings`)[cite: 4].
* Живой математический счетчик токенов (Эвристика: ~2.5 символа = 1 токен) в нижнем Status Bar[cite: 4].
* Отправка запросов по `Ctrl + Enter` (обычный `Enter` работает как перенос строки)[cite: 4].
* Умное масштабирование окна при запуске (адаптация под размер экрана)[cite: 4].

**Фаза 7: Встроенный Файловый Менеджер (Завершено ✅)**
* Мини-панель над деревом проекта с кнопками "📄 Создать файл", "📁 Создать папку", "✏️ Переименовать", "🗑️ Удалить"[cite: 4].
* Безопасное удаление с запросом подтверждения через системное диалоговое окно[cite: 4].

**Фаза 8: Навигация и Вкладки (Tabs) (Завершено ✅)**
* Внедрение `QTabWidget` над окном редактора кода[cite: 4].
* Отображение имени текущего активного файла (например, `📄 main.py`), что исключает путаницу[cite: 4].

**Фаза 9: Умное тегирование и Контекст (Drag & Drop + ПКМ) (Завершено ✅)**
* Возможность перетаскивать файлов из дерева прямо в чат (Drag&Drop)[cite: 4].
* Автоматическая вставка тегов в формате `@filename.py` (или `@[file name.txt]` для имен с пробелами)[cite: 4].
* Интеграция контекстного меню (ПКМ) по тегам в чате для опции «📎 Прикрепить код» (выделяется жирным зеленым) или открепления[cite: 4].
* Поддержка мульти-выделения (Shift/Ctrl) файлов в дереве с возможностью пакетной отправки их кода в чат через ПКМ[cite: 4].

**Фаза 10: Управление проектами и Security (Завершено ✅)**
* Кнопка `📂` для быстрого переключения между разными папками-проектами[cite: 4]. Авто-сохранение последнего проекта[cite: 4].
* Защита от Path Traversal: программа блокирует попытки ИИ изменить файлы за пределами директории проекта[cite: 4].
* Добавлена опция просмотра "сырого" промпта по ссылке в чате[cite: 4].

**Фаза 11: Архитектурный рефакторинг (Завершено ✅)**
* Разделение монолитного `ui_main.py` на логические изолированные модули по принципу Single Responsibility[cite: 4].
* Выделение `custom_widgets.py` (умные текстовые поля, Drag&Drop, подсветка синтаксиса)[cite: 4].
* Выделение `file_explorer.py` (управление деревом файлов, выбор проектов, контекстное меню)[cite: 4].
* Значительное упрощение главного класса `MainWindow` для дальнейшего масштабирования IDE[cite: 4].

**Фаза 12: Локальная Система памяти (Smart Memory & Time Machine) (Завершено ✅)**
* Изолированная история проектов: папка `.vibecoder/` в корне каждого проекта для логов и бэкапов[cite: 4].
* Умная история сессий (History Dialog): разделение на список запросов и деталей (Master-Detail UI), быстрый текстовый поиск по вопросам и мыслям ИИ[cite: 4].
* Локальная Машина Времени: автоматическое сохранение 10 последних версий каждого файла при утверждении кода (уровень приложения)[cite: 4].
* UI отката версий: визуальное сравнение (Diff) старых бэкапов с текущим кодом и кнопка восстановления[cite: 4].

**Фаза 13: Умный патчинг и Авто-контекст (Smart Diff & Tool Calling) (Завершено ✅)**
* Smart Diff: ИИ больше не присылает полные файлы, а использует команды поиска и замены (`search`/`replace`), экономя токены и время[cite: 4].
* Конвейер Ревью: возможность ИИ изменять сразу несколько файлов[cite: 4]. Программа открывает окна Diff поочередно для безопасного утверждения[cite: 4].
* Инструментальный вызов (Auto-context): ИИ может запрашивать файлы из дерева проекта, а программа (после подтверждения пользователя) автоматически извлекает их код и отправляет ИИ[cite: 4].
* Переход на математически точные Относительные Пути для всех файлов и тегов (нормализация слешей Windows/Unix)[cite: 4].

**Фаза 14: Локальное Управление версиями (Smart Git) (Завершено ✅)**
* Интеграция локального Git через `subprocess` напрямую в интерфейс IDE[cite: 4].
* Динамическая кнопка `📦 Git` на панели управления (показывает количество измененных файлов и меняет цвет)[cite: 4].
* Инициализация пустого репозитория прямо из программы[cite: 4].
* Генерация описания коммитов (AI Commits): VibeCoder сам собирает `git diff` и отправляет его в ИИ для написания логов коммита[cite: 4].
* Бронебойный парсинг статусов Git (`--porcelain`): умное разделение строк, предотвращающее обрезку первых букв папок и файлов (поддержка пробелов и переименований путей).

**Фаза 15: Интеллектуальная структура и Smart .gitignore (Завершено ✅)**
* Чтение и обработка файла `.gitignore` на лету с помощью кастомной модели `QFileSystemModel`[cite: 4].
* Визуальное затухание (темно-серый цвет текста) для мусорных и игнорируемых папок/файлов в левой панели[cite: 4].
* Интеграция контекстного меню (ПКМ) в дереве файлов для быстрого добавления или удаления путей из `.gitignore`[cite: 4].
* Обучение ИИ самостоятельно запрашивать создание новых файлов и структуры папок через поле `create_files` в JSON[cite: 4].
* Всплывающее UI-окно с чекбоксами для финального подтверждения списка создаваемых файлов пользователем[cite: 4].

**Фаза 16: Мульти-маршрутизация и Heartbeat (Tab Routing & Anti-Jump) (Завершено ✅)**
* Уникальная идентификация (ID) для каждой открытой вкладки Gemini (сохраняется в `sessionStorage`)[cite: 4].
* Система Heartbeat (Пульс): вкладки каждые 2 секунды сообщают серверу свой ID и назначенное имя[cite: 4].
* Умное кеширование (Anti-Jump): сервер `bridge.py` сохраняет вкладки в памяти с 12-секундным таймаутом, полностью предотвращая "прыжки" UI интерфейса во время микро-зависаний браузера при генерации ответа ИИ.
* Компактная UI-плашка (VibeBadge) на странице Gemini с функцией Drag & Drop (перемещение за иконку ⠿ с сохранением координат)[cite: 4].
* Переименование вкладок по клику на имя прямо в плашке браузера[cite: 4].
* Выпадающий список в статус-баре IDE для выбора активной вкладки и целевая маршрутизация задач (target_id), жестко привязанная к ID сессии[cite: 4].

**Фаза 17: Двусторонняя синхронизация с GitHub и Pro-Git (Завершено ✅)**
* Привязка удаленного репозитория (Remote origin) и двусторонняя синхронизация: отправка кода (Push) и скачивание (Pull)[cite: 4].
* Точечные коммиты (Staging Area): замена плоского списка на иерархическое дерево папок (`QTreeWidget`) с умными QSS-галочками (выбор папки выделяет все файлы внутри)[cite: 4].
* Хирургическое извлечение (Surgical Pull): возможность скачать с GitHub не весь проект, а только выбранные галочками файлы (`git fetch` + `git checkout`)[cite: 4].
* Фильтр файлов: чекбокс для переключения между видом "Только измененные файлы" (для коммитов) и "Показать все файлы проекта" (для извлечения из облака)[cite: 4]. Умное скрытие папок для предотвращения зависаний интерфейса[cite: 4].
* Облачная Машина Времени (Git Timeline): просмотр истории коммитов конкретного файла через ПКМ в дереве Git[cite: 4].
* Встроенный просмотрщик патчей (Diff Viewer): отображение посимвольной разницы (красный/зеленый текст с HTML-рендером) и полного сырого кода для любого старого коммита во вкладках (`QTabWidget`)[cite: 4]. Откат локального файла до облачной версии в один клик[cite: 4].

## 6. Управление и Горячие клавиши
* `Ctrl + Enter`: Мгновенная отправка задачи из поля ввода[cite: 4].
* `Enter`: Перенос строки в поле ввода[cite: 4].
* `Ctrl + S`: Ручное быстрое сохранение открытого файла (с авто-созданием бэкапа в локальной Машине Времени)[cite: 4].
* `Ctrl + Колесико мыши`: Изменение размера шрифта (поле ввода, логи чата, редактор кода)[cite: 4].
* `Клик по кнопке 📦 Git`: Открывает интерфейс просмотра статуса, управления коммитами и облачной синхронизации[cite: 4].
* `Клик по ссылке [Показать текст] / [Показать сырой промпт]`: Просмотр скрытого системного промпта[cite: 4].
* `Drag & Drop файлов`: Перетаскивание файлов из дерева в текстовое поле создает теги упоминаний (`@[core/main.py]`)[cite: 4].
* `ПКМ по тегу @файл`: Прикрепляет/открепляет исходный код файла к запросу (цвет меняется на зеленый)[cite: 4].
* `ПКМ по файлу в дереве проекта -> 🕒 Машина Времени`: Открывает интерфейс отката старых версий файла из папки `.vibe_backups/`[cite: 4].
* `ПКМ по файлу в окне Git -> 🕒 Посмотреть Git-историю файла`: Открывает окно просмотра коммитов, сравнения Diff и отката к старой версии из облака[cite: 4].
* `ПКМ по файлу в дереве проекта -> 🚫 Добавить в .gitignore`: Автоматически прописывает путь в игнор-лист[cite: 4].
* `Shift/Ctrl + Клик в дереве проекта`: Мульти-выделение файлов для пакетных операций[cite: 4].
* `Левая кнопка мыши (ЛКМ) по иконке ⠿ на плашке в браузере:` Плавное перетаскивание статусной панели по экрану[cite: 4].
* `ЛКМ по имени в плашке браузера:` Вызов диалогового окна для смены человекочитаемого имени вкладки[cite: 4].
