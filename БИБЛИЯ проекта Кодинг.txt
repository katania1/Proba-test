# 📘 VibeCoder v1.21 — Project Bible (Ultimate MVC + Smart RAG + Custom API)[cite: 1]

## 1. Концепция проекта[cite: 1]
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM)[cite: 1].
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (Semantic RAG, локальный Git, терминал)[cite: 1].

## 2. Технический стек[cite: 1]
* **Язык:** Python 3.10+[cite: 1]
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)[cite: 1]
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)[cite: 1]
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)[cite: 1]
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini[cite: 1]
* **База данных и Память:** Локальные JSON-файлы (логи) + Локальная векторная БД (RAG)[cite: 1]
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash)[cite: 1].

**Фаза 24: Интеграция MCP (Model Context Protocol) и Агентного цикла (Завершено ✅)**
* Внедрен локальный диспетчер инструментов `mcp_manager.py` (Model Context Protocol).
* Поддержка внешних MCP-серверов по стандарту JSON-RPC (stdio) для интеграции Node.js пакетов, таких как `@upstash/context7-mcp` для получения свежей документации фреймворков.
* Разработаны встроенные локальные инструменты: бронебойный веб-поиск через DuckDuckGo Lite с обходом блокировок и выполнение SQL-запросов (`SELECT`) к локальным БД SQLite.
* Реализован цикл Агента (Agentic Loop) в `api_worker.py`: ИИ имеет лимит до 5 последовательных шагов на вызов инструментов для проведения расследования перед выдачей финального ответа.
* Внедрен отказоустойчивый парсер `_extract_first_json` для решения проблемы конфликта двойных JSON-ответов от ИИ (разделение команд инструмента и Оркестратора).

**Фаза 25: Гибридный режим (Markdown + Кодинг) и Умный UI (Завершено ✅)**
* Внедрен "Гибридный режим" Оркестратора: IDE понимает разницу между абстрактным вопросом пользователя и задачей на изменение файлов. ИИ теперь может отвечать обычным текстом, не ломая парсер.
* Разработан кастомный HTML-рендер Markdown (с сохранением темной темы и подсветкой синтаксиса), интегрированный прямо в виджет чата.
* Внедрена функция Reverse-Engineering HTML -> Markdown в расширении Chrome, позволяющая извлекать чистый код из "красивых" блоков Gemini для корректного отображения в VibeCoder.
* Умная обработка лимитов: при достижении квоты Pro-версии, VibeCoder предлагает выбор — запустить авто-сборку транзитного пакета или принудительно (ignoreLimitsSession) продолжить работу на быстрой Flash-модели без блокировок.
* Жесткий механизм авто-исправления JSON: парсер теперь напрямую указывает ИИ на забытые экранирования двойных кавычек в блоках кода, обучая его на лету.
