

# 📘 VibeCoder v1.23 — Project Bible (Ultimate MVC + Smart RAG + Custom API)

## 1. Концепция проекта

**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM).
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (Semantic RAG, локальный Git, терминал, виртуальная файловая система).

## 2. Технический стек

* **Язык:** Python 3.10+
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini, Virtual File System (Base64 -> DOM File Drop).
* **База данных и Память:** Локальные JSON-файлы (логи) + Локальная векторная БД ChromaDB (RAG)
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash).

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**

* Профессиональная темная тема VS Code.
* Кастомный QTreeView (Дерево файлов) без лишних заголовков.
* Подсветка синтаксиса Python.
* **Pro-Навигация (Стиль Notepad++):** Внедрены жесткие пунктирные направляющие отступов (`LookBoth`) и система сплошных светлых нитей сворачивания блоков (`BoxedTree`), обеспечивающие идеальную визуальную иерархию кода.
* Исправление конфликта окончаний строк (LF/CRLF), исключающее появление пустых строк при редактировании в сторонних IDE.

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**

* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ.
* Ультимативный системный промпт: принудительный JSON, запрет заглушек.
* Автоматический Валидатор: если ИИ ошибается (ломает JSON), скрипт САМ отправляет гневное сообщение и заставляет ИИ переписать код.
* Бронебойный экстрактор текста (Robust Parser) для сломанных JSON-ответов (теперь вынесен в `ai_orchestrator.py`).

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**

* Интеграция локальной векторной базы ChromaDB для семантического поиска.
* Окно Аналитики RAG: отслеживание лимитов API, сортировка файлов.
* **Бронебойный Поллинг (Polling Engine):** Система автоматически сканирует файлы каждые 10 секунд и обновляет индекс, если файл был изменен сторонним редактором.

**Фаза 21: Интеграция MCP (Model Context Protocol) и Агентного цикла (Завершено ✅)**

* Внедрен локальный диспетчер инструментов `mcp_manager.py`.
* Поддержка внешних MCP-серверов по стандарту JSON-RPC (stdio).
* Локальные инструменты: бронебойный веб-поиск (DuckDuckGo Lite) и чтение БД SQLite.
* Цикл Агента (Agentic Loop): ИИ имеет лимит до 5 последовательных шагов на вызов инструментов перед выдачей финального кода.

**Фаза 22: Гибридный режим (Markdown + Кодинг) (Завершено ✅)**

* Кастомный HTML-рендер Markdown (с сохранением темной темы и подсветкой синтаксиса), интегрированный прямо в `VibeChatBrowser`.
* Reverse-Engineering HTML -> Markdown в расширении Chrome.

**Фаза 23: Двухкнопочный UI и Виртуальная файловая система (VFS) (Завершено ✅)**

* **Двухкнопочная маршрутизация:** Полное визуальное и логическое разделение на `[ 💬 Спросить ]` (отправка легкого текста без RAG) и `[ ⚡ Кодить ]` (полный RAG, сборка архитектуры, требование JSON). Убраны лишние тумблеры и чекбоксы.
* **Виртуальная файловая система (VFS):** Тотальное решение проблемы зависания веб-интерфейса браузера. IDE упаковывает тяжелый RAG-контекст, структуру папок и жесткие правила (`vibe_instructions.txt`) в виртуальные файлы, кодирует их в Base64 и передает через мост.
* **Double-Tap Injection:** Расширение Chrome аппаратно симулирует вставку файлов в DOM-дерево Gemini (`input[type="file"]` или хак через `ClipboardEvent('paste')`), обходя ограничения безопасности браузера.

**Фаза 24: Архитектурный рефакторинг "Убийство Божественного Объекта" (Завершено ✅)**

* Внедрен паттерн "Строитель" (`ContextBuilder`): вынесена грязная логика сборки Base64 картинок, RAG-запросов и парсинга тегов из контроллера ИИ.
* Распил `ai_controller.py`: логика работы с БД ушла в `rag_controller.py`, а парсинг грязного JSON и рендеринг HTML ушли в `ai_orchestrator.py`. `ai_controller.py` стал чистым диспетчером потоков (Router).
* Инъекция диффов: починена логика генерации ИИ-коммитов путем переноса текста из виртуального `.txt` файла напрямую в текстовое тело промпта, чтобы обойти баги чтения файлов в новых чатах Gemini.

## 6. Управление и Горячие клавиши

* `Enter`: Отправка "Легкого" вопроса в чат (Режим "💬 Спросить" - без RAG, парсинга кода и сброса структуры).
* `Ctrl + Enter`: Отправка "Тяжелой" задачи на кодинг (Режим "⚡ Кодить" - с RAG, контекстом файлов, JSON-Оркестратором и окном ревью).
* `Shift + Enter`: Перенос строки в поле ввода (стандартное поведение для многострочного текста).
* `Ctrl + S`: Ручное сохранение файла в редакторе (автоматически создает локальный бэкап в папке `.vibecoder`).
* `Ctrl + ~`: Показать/Скрыть нижнюю панель встроенного терминала.
* `Стрелки Вверх/Вниз (в терминале)`: Навигация по истории введенных команд.
* `Ctrl + Колесико мыши`: Независимое изменение размера шрифта интерфейса (дерево файлов, редактор, окно логов).
* `Автоматическое RAG-обновление`: При сохранении файла в любом стороннем редакторе (VS Code, Notepad++), IDE сама перестроит векторный индекс измененного файла в течение 10 секунд (Polling Engine).
* `Клик по кнопке 📦 Git`: Открывает окно синхронизации с облаком и точечную Staging Area.
* `Drag & Drop файлов`: Быстрое создание тегов упоминаний (`@[core/main.py]`) из дерева проекта в чат.
* `ПКМ по тегу @файл`: Вызов меню для прикрепления/открепления исходного кода файла напрямую к промпту.
* `ПКМ по файлу в дереве -> 🕒 Машина Времени`: Моментальный визуальный откат файла из истории локальных бэкапов.
* `ПКМ по файлу в окне Git -> 🕒 Посмотреть Git-историю`: Откат файла до конкретного облачного коммита с GitHub.
* `ПКМ по файлу в дереве -> 🚫 Добавить в .gitignore`: Динамически прописать выбранный путь в список игнорируемых файлов.
* `Shift/Ctrl + Клик в дереве файлов`: Стандартное мульти-выделение для пакетной привязки.
* `ПКМ по модели в окне настроек API:` Добавление кастомного текстового примечания к модели.
* `Клик по имени проекта (статус над деревом файлов):` Открывает визуальный Менеджер Проектов для смены рабочей директории.
