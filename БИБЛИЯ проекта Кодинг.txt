

# 📘 VibeCoder v1.19 — Project Bible (Ultimate MVC Edition + Terminal + Custom API)

## 1. Концепция проекта
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM)[cite: 27].
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (горячие клавиши, зум, статистика токенов, встроенный терминал)[cite: 27].

## 2. Технический стек
* **Язык:** Python 3.10+[cite: 27]
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)[cite: 27]
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)[cite: 27]
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)[cite: 27]
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini[cite: 27]
* **База данных:** Локальные JSON-файлы (логирование истории чатов)[cite: 27]
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash).[cite: 27]

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**
* Профессиональная темная тема VS Code.[cite: 27]
* Кастомный QTreeView (Дерево файлов) без лишних заголовков.[cite: 27]
* Подсветка синтаксиса Python.[cite: 27]
* Глобальные CSS-стили: стилизованные разделители панелей (QSplitter) и темные всплывающие подсказки (QToolTip).[cite: 27]

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**
* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ.[cite: 27]
* Ультимативный системный промпт: принудительный JSON, запрет заглушек.[cite: 27]
* Автоматический Валидатор: если ИИ ошибается (ломает JSON или присылает ленивый код), скрипт САМ отправляет гневное сообщение с оригинальным промптом и заставляет ИИ переписать код (до 2-х попыток).[cite: 27]
* Бронебойный экстрактор текста (Robust Parser): способность извлекать мысли ИИ и коммиты даже из сломанного JSON-ответа.[cite: 27]

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**
* Обмен данными через `localhost:5070`.[cite: 27]
* Единый контекст сессии: ИИ больше не сбрасывается после каждого ответа, диалог идет в одной ветке.[cite: 27]
* Детектор лимитов: расширение читает уведомления Gemini и статус кнопки "Pro". Если лимиты исчерпаны, UI автоматически ставится на паузу и выдает кастомное темное предупреждение (`show_popup`).[cite: 27]

**Фаза 4: Система Ревью (Diff Viewer) (Завершено ✅)**
* Live Preview: присланный ИИ код мгновенно отображается в редакторе (до сохранения).[cite: 27]
* Модуль сравнения старого и нового кода (difflib) при нажатии "✅ Утвердить".[cite: 27]
* Автоматическое создание бэкапов (.bak) перед любой перезаписью файла.[cite: 27]

**Фаза 5: Логирование и Эстафета (Завершено ✅)**

* Кнопка "🔄 Эстафета": формирует стартовый пакет, передает флаг `is_relay` в мост, после чего расширение само открывает новый чат Gemini.[cite: 27]

**Фаза 12: Локальная Система памяти (Smart Memory & Time Machine) (Завершено ✅)**

* Умная история сессий (History Dialog): разделение на список запросов и деталей (Master-Detail UI), быстрый текстовый поиск по вопросам и мыслям ИИ.[cite: 27]
* Локальная Машина Времени: автоматическое сохранение 10 последних версий каждого файла при утверждении кода (уровень приложения).[cite: 27]
* UI отката версий: визуальное сравнение (Diff) старых бэкапов с текущим кодом и кнопка восстановления.[cite: 27]

**Фаза 13: Умный патчинг и Авто-контекст (Smart Diff & Tool Calling) (Завершено ✅)**
* Smart Diff: ИИ больше не присылает полные файлы, а использует команды поиска и замены (`search`/`replace`), экономя токены и время.[cite: 27]
* Конвейер Ревью: возможность ИИ изменять сразу несколько файлов. Программа открывает окна Diff поочередно для безопасного утверждения.[cite: 27]
* Инструментальный вызов (Auto-context): ИИ может запрашивать файлы из дерева проекта, а программа (после подтверждения пользователя) автоматически извлекает их код и отправляет ИИ.[cite: 27]
* Переход на математически точные Относительные Пути для всех файлов и тегов (нормализация слешей Windows/Unix).[cite: 27]

**Фаза 14: Локальное Управление версиями (Smart Git) (Завершено ✅)**
* Интеграция локального Git через `subprocess` напрямую в интерфейс IDE.[cite: 27]
* Динамическая кнопка `📦 Git` на панели управления (показывает количество измененных файлов и меняет цвет).[cite: 27]
* Инициализация пустого репозитория прямо из программы.[cite: 27]
* Генерация описания коммитов (AI Commits): VibeCoder сам собирает `git diff` и отправляет его в ИИ для написания логов коммита.[cite: 27]
* Бронебойный парсинг статусов Git (`--porcelain`): умное разделение строк, предотвращающее обрезку первых букв папок и файлов (поддержка пробелов и переименований путей).[cite: 27]

**Фаза 15: Интеллектуальная структура и Smart .gitignore (Завершено ✅)**
* Чтение и обработка файла `.gitignore` на лету с помощью кастомной модели `QFileSystemModel`.[cite: 27]
* Визуальное затухание (темно-серый цвет текста) для мусорных и игнорируемых папок/файлов в левой панели.[cite: 27]
* Интеграция контекстного меню (ПКМ) в дереве файлов для быстрого добавления или удаления путей из `.gitignore`.[cite: 27]
* Обучение ИИ самостоятельно запрашивать создание новых файлов и структуры папок через поле `create_files` в JSON.[cite: 27]
* Всплывающее UI-окно с чекбоксами для финального подтверждения списка создаваемых файлов пользователем.[cite: 27]

**Фаза 16: Мульти-маршрутизация и Heartbeat (Tab Routing & Anti-Jump) (Завершено ✅)**
* Уникальная идентификация (ID) для каждой открытой вкладки Gemini (сохраняется в `sessionStorage`).[cite: 27]
* Система Heartbeat (Пульс): вкладки каждые 2 секунды сообщают серверу свой ID и назначенное имя.[cite: 27]
* Умное кеширование (Anti-Jump): сервер `bridge.py` сохраняет вкладки в памяти с 12-секундным таймаутом, полностью предотвращая "прыжки" UI интерфейса во время микро-зависаний браузера при генерации ответа ИИ.[cite: 27]
* Компактная UI-плашка (VibeBadge) на странице Gemini с функцией Drag & Drop (перемещение за иконку ⠿ с сохранением координат).[cite: 27]
* Переименование вкладок по клику на имя прямо в плашке браузера.[cite: 27]
* Выпадающий список в статус-баре IDE для выбора активной вкладки и целевая маршрутизация задач (target_id), жестко привязанная к ID сессии.[cite: 27]

**Фаза 17: Двусторонняя синхронизация с GitHub и Pro-Git (Завершено ✅)**
* Привязка удаленного репозитория (Remote origin) и двусторонняя синхронизация: отправка кода (Push) и скачивание (Pull).[cite: 27]
* Точечные коммиты (Staging Area): замена плоского списка на иерархическое дерево папок (`QTreeWidget`) с умными QSS-галочками (выбор папки выделяет все файлы внутри).[cite: 27]
* Хирургическое извлечение (Surgical Pull): возможность скачать с GitHub не весь проект, а только выбранные галочками файлы (`git fetch` + `git checkout`).[cite: 27]
* Фильтр файлов: чекбокс для переключения между видом "Только измененные файлы" (для коммитов) и "Показать все файлы проекта" (для извлечения из облака). Умное скрытие папок для предотвращения зависаний интерфейса.[cite: 27]
* Облачная Машина Времени (Git Timeline): просмотр истории коммитов конкретного файла через ПКМ в дереве Git.[cite: 27]
* Встроенный просмотрщик патчей (Diff Viewer): отображение посимвольной разницы (красный/зеленый текст с HTML-рендером) и полного сырого кода для любого старого коммита во вкладках (`QTabWidget`). Откат локального файла до облачной версии в один клик.[cite: 27]

**Фаза 18: Продвинутая интеграция API-провайдеров (Multi-Engine & Custom API) (Завершено ✅)**
* Реализация паттерна "Стратегия" (`AbstractProvider`) для поддержки прямого подключения к OpenAI, Anthropic и Gemini API.[cite: 27]
* Система Custom API: возможность добавлять любые OpenAI-совместимые сети (OpenRouter, Groq, Ollama, LM Studio) через динамические вкладки.[cite: 27]
* Пакетная проверка моделей (Batch Verification): умная очередь пингования моделей с задержкой 1.5 сек (защита от Rate Limit). Автоматическая подмена `max_tokens` на `max_completion_tokens` для o1-совместимых рассуждающих моделей.[cite: 27]
* Визуальное управление списком: чекбоксы для показа/скрытия моделей в главном селекторе, сохранение статусов (`✅`/`❌`).[cite: 27]
* Система примечаний (Notes): контекстное меню (ПКМ) для добавления личных заметок к моделям ("Хорошо пишет", "Галлюцинирует") и "живой" поиск по списку.[cite: 27]
* Умная маршрутизация задач (`AIController`): селектор в статус-баре направляет промпт либо в браузерный мост, либо напрямую в API, автоматически подставляя нужный `Base URL` и ключ. Асинхронное выполнение через `APIWorker`.[cite: 27]

**Фаза 19: Встроенный Интеллектуальный Терминал (Integrated Terminal) (Завершено ✅)**
* Интеграция системной командной строки (cmd.exe) прямо под редактором кода через модуль `QProcess`.[cite: 27]
* Умная кодировка (CP866 -> UTF-8): полная поддержка русского языка и путей с пробелами и кириллицей в Windows-системах.[cite: 27]
* Разделенный UI (Split UX): большое текстовое окно "только-для-чтения" для логов и отдельная строка `QLineEdit` для ввода с поддержкой истории команд (стрелки Вверх/Вниз).[cite: 27]
* Авто-активация окружения: при запуске терминал автоматически ищет и незаметно запускает `venv/Scripts/activate.bat`, подготавливая среду к установке пакетов (`pip install`).[cite: 27]
* Очистка от ANSI-мусора: регулярные выражения фильтруют цветовые и управляющие коды терминала, сохраняя чистый текст в логах.[cite: 27]
* Глобальный шорткат (`Ctrl + ~`) для быстрого скрытия и открытия панели терминала. Перезапуск процесса при смене директории проекта.[cite: 27]

**Фаза 20: Многопроектная архитектура и Портативный режим (Завершено ✅)**
* Полный отказ от Системного Реестра Windows в пользу локальных `.ini` файлов конфигурации (Portable Mode).
* Встроенный Менеджер Проектов с визуальным интерфейсом для быстрого создания и переключения между проектами с сохранением истории.
* Интеллектуальный сброс контекста: полная изоляция чатов, открытых файлов и кэша ИИ при переключении между рабочими областями.
* Disaster Recovery (Авто-восстановление): программа автоматически определяет отсутствие среды `venv` при скачивании проекта с GitHub и предлагает самостоятельно ее воссоздать и установить зависимости через встроенный терминал.

## 5. Дорожная карта (Roadmap: Конкуренция с ведущими AI-IDE)
* **Фаза 21: Аналитика токенов.** Создать систему графиков и статистики затрат на основе логов ChatLogger (вкладка в «Машине Времени»)[cite: 27].
* **Фаза 22: RAG-индексация кодовой базы (Semantic Code Search).** Интеграция локальной векторной базы (например, ChromaDB или FAISS) для эмбеддинга всего проекта. Позволит ИИ самостоятельно искать релевантный код по всему проекту без необходимости ручного прикрепления файлов пользователем (как в Cursor / Windsurf).
* **Фаза 23: Агентный режим отладки (Auto-healing & Linter Integration).** Интеграция с локальными линтерами (flake8, pylint) и автоматический запуск тестов в фоновом терминале. Перехват Traceback ошибок и автоматическая отправка их ИИ для самостоятельного анализа и исправления (цикл автономной отладки).
* **Фаза 24: Интеграция MCP (Model Context Protocol).** Добавление поддержки стандарта MCP, что позволит ИИ-движку VibeCoder безопасно вызывать внешние инструменты: искать решения на StackOverflow через веб-поиск, читать свежую документацию фреймворков и делать запросы к базам данных SQLite/PostgreSQL напрямую из чата.
* **Фаза 25: Интеллектуальный Визуализатор Архитектуры (Code Map).** Динамическое построение UML-диаграмм и графов зависимостей файлов проекта. Поможет как пользователю для навигации, так и ИИ для лучшего понимания связей между модулями сложной системы.

## 6. Управление и Горячие клавиши
* `Ctrl + Enter`: Мгновенная отправка задачи из поля ввода.[cite: 27]
* `Enter`: Перенос строки в поле ввода.[cite: 27]
* `Ctrl + S`: Ручное быстрое сохранение открытого файла (с авто-созданием бэкапа в локальной Машине Времени).[cite: 27]
* `Ctrl + ~` (или `Ctrl + \``): Показать/Скрыть нижнюю панель терминала.[cite: 27]
* `Стрелки Вверх/Вниз (в терминале)`: Навигация по истории введенных команд.[cite: 27]
* `Ctrl + Колесико мыши`: Изменение размера шрифта (поле ввода, логи чата, редактор кода).[cite: 27]
* `Клик по кнопке 📦 Git`: Открывает интерфейс просмотра статуса, управления коммитами и облачной синхронизации.[cite: 27]
* `Клик по ссылке [Показать текст] / [Показать сырой промпт]`: Просмотр скрытого системного промпта.[cite: 27]
* `Drag & Drop файлов`: Перетаскивание файлов из дерева в текстовое поле создает теги упоминаний (`@[core/main.py]`).[cite: 27]
* `ПКМ по тегу @файл`: Прикрепляет/открепляет исходный код файла к запросу (цвет меняется на зеленый).[cite: 27]
* `ПКМ по файлу в дереве проекта -> 🕒 Машина Времени`: Открывает интерфейс отката старых версий файла из папки `.vibe_backups/`.[cite: 27]
* `ПКМ по файлу в окне Git -> 🕒 Посмотреть Git-историю файла`: Открывает окно просмотра коммитов, сравнения Diff и отката к старой версии из облака.[cite: 27]
* `ПКМ по файлу в дереве проекта -> 🚫 Добавить в .gitignore`: Автоматически прописывает путь в игнор-лист.[cite: 27]
* `Shift/Ctrl + Клик в дереве проекта`: Мульти-выделение файлов для пакетных операций.[cite: 27]
* `Левая кнопка мыши (ЛКМ) по иконке ⠿ на плашке в браузере:` Плавное перетаскивание статусной панели по экрану.[cite: 27]
* `ЛКМ по имени в плашке браузера:` Вызов диалогового окна для смены человекочитаемого имени вкладки.[cite: 27]
* `ПКМ по модели в окне API:` Добавление текстового примечания к модели, копирование текста ошибки или принудительная перепроверка.[cite: 27]
* `ПКМ по названию вкладки провайдера (в окне API):` Смена человекочитаемого имени кастомного провайдера (например, с "custom_1" на "OpenRouter").[cite: 27]
* `Клик по имени проекта (над деревом файлов):` Открывает Менеджер Проектов для быстрого создания, открытия и переключения между рабочими директориями.
