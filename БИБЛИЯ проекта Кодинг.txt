# 📘 VibeCoder v1.19 — Project Bible (Ultimate MVC Edition + Terminal + Custom API)

## 1. Концепция проекта
**VibeCoder** — это локальная IDE и умный AI-оркестратор профессионального уровня для "вайбкодинга" через веб-версию Gemini 3.1 Pro и прямые API-запросы (OpenAI, Anthropic, Gemini API, а также любые кастомные OpenAI-совместимые сети вроде OpenRouter, Groq и локальных LLM).
Программа работает как невидимый посредник, обеспечивая бесшовную генерацию кода, жесткий контроль над галлюцинациями ИИ, сохранение единого контекста сессии и удобную среду с умной эргономикой (горячие клавиши, зум, статистика токенов, встроенный терминал).

## 2. Технический стек
* **Язык:** Python 3.10+
* **GUI-фреймворк:** PyQt6 (интерфейс) + QSettings (память состояний)
* **Редактор кода:** PyQt6-QScintilla (движок Scintilla / Notepad++)
* **Backend:** Flask (порт 5070) + Flask-CORS (Транспортный мост)
* **Browser:** Chrome Extension (Manifest v3) — инъекция JS в DOM-дерево Gemini
* **База данных:** Локальные JSON-файлы (логирование истории чатов)
* **Системная интеграция:** QProcess для встроенного эмулятора терминала (cmd/bash).

## 4. Что реализовано (Текущий статус)

**Фаза 1: Дизайн и Базовый Редактор (Завершено ✅)**
* Профессиональная темная тема VS Code.
* Кастомный QTreeView (Дерево файлов) без лишних заголовков.
* Подсветка синтаксиса Python.
* Глобальные CSS-стили: стилизованные разделители панелей (QSplitter) и темные всплывающие подсказки (QToolTip).

**Фаза 2: Оркестратор и Валидация (Завершено ✅)**
* Система "Project Bible": автоматический сбор структуры папок для контекста ИИ.
* Ультимативный системный промпт: принудительный JSON, запрет заглушек.
* Автоматический Валидатор: если ИИ ошибается (ломает JSON или присылает ленивый код), скрипт САМ отправляет гневное сообщение с оригинальным промптом и заставляет ИИ переписать код (до 2-х попыток).
* Бронебойный экстрактор текста (Robust Parser): способность извлекать мысли ИИ и коммиты даже из сломанного JSON-ответа.

**Фаза 3: Транспортный Мост и Память Контекста (Завершено ✅)**
* Обмен данными через `localhost:5070`.
* Единый контекст сессии: ИИ больше не сбрасывается после каждого ответа, диалог идет в одной ветке.
* Детектор лимитов: расширение читает уведомления Gemini и статус кнопки "Pro". Если лимиты исчерпаны, UI автоматически ставится на паузу и выдает кастомное темное предупреждение (`show_popup`).

* Инструментальный вызов (Auto-context): ИИ может запрашивать файлы из дерева проекта, а программа (после подтверждения пользователя) автоматически извлекает их код и отправляет ИИ.

* Интеграция локального Git через `subprocess` напрямую в интерфейс IDE.
* Динамическая кнопка `📦 Git` на панели управления (показывает количество измененных файлов и меняет цвет).
* Инициализация пустого репозитория прямо из программы.
* Генерация описания коммитов (AI Commits): VibeCoder сам собирает `git diff` и отправляет его в ИИ для написания логов коммита.
* Бронебойный парсинг статусов Git (`--porcelain`): умное разделение строк, предотвращающее обрезку первых букв папок и файлов (поддержка пробелов и переименований путей).

**Фаза 15: Интеллектуальная структура и Smart .gitignore (Завершено ✅)**
* Чтение и обработка файла `.gitignore` на лету с помощью кастомной модели `QFileSystemModel`.
* Визуальное затухание (темно-серый цвет текста) для мусорных и игнорируемых папок/файлов в левой панели.
* Интеграция контекстного меню (ПКМ) в дереве файлов для быстрого добавления или удаления путей из `.gitignore`.
* Обучение ИИ самостоятельно запрашивать создание новых файлов и структуры папок через поле `create_files` в JSON.
* Всплывающее UI-окно с чекбоксами для финального подтверждения списка создаваемых файлов пользователем.

**Фаза 16: Мульти-маршрутизация и Heartbeat (Tab Routing & Anti-Jump) (Завершено ✅)**
* Уникальная идентификация (ID) для каждой открытой вкладки Gemini (сохраняется в `sessionStorage`).
* Система Heartbeat (Пульс): вкладки каждые 2 секунды сообщают серверу свой ID и назначенное имя.
* Умное кеширование (Anti-Jump): сервер `bridge.py` сохраняет вкладки в памяти с 12-секундным таймаутом, полностью предотвращая "прыжки" UI интерфейса во время микро-зависаний браузера при генерации ответа ИИ.
* Компактная UI-плашка (VibeBadge) на странице Gemini с функцией Drag & Drop (перемещение за иконку ⠿ с сохранением координат).
* Переименование вкладок по клику на имя прямо в плашке браузера.
* Выпадающий список в статус-баре IDE для выбора активной вкладки и целевая маршрутизация задач (target_id), жестко привязанная к ID сессии.

**Фаза 17: Двусторонняя синхронизация с GitHub и Pro-Git (Завершено ✅)**
* Привязка удаленного репозитория (Remote origin) и двусторонняя синхронизация: отправка кода (Push) и скачивание (Pull).
* Точечные коммиты (Staging Area): замена плоского списка на иерархическое дерево папок (`QTreeWidget`) с умными QSS-галочками (выбор папки выделяет все файлы внутри).
* Хирургическое извлечение (Surgical Pull): возможность скачать с GitHub не весь проект, а только выбранные галочками файлы (`git fetch` + `git checkout`).
* Фильтр файлов: чекбокс для переключения между видом "Только измененные файлы" (для коммитов) и "Показать все файлы проекта" (для извлечения из облака). Умное скрытие папок для предотвращения зависаний интерфейса.
* Облачная Машина Времени (Git Timeline): просмотр истории коммитов конкретного файла через ПКМ в дереве Git.
* Встроенный просмотрщик патчей (Diff Viewer): отображение посимвольной разницы (красный/зеленый текст с HTML-рендером) и полного сырого кода для любого старого коммита во вкладках (`QTabWidget`). Откат локального файла до облачной версии в один клик.

**Фаза 18: Продвинутая интеграция API-провайдеров (Multi-Engine & Custom API) (Завершено ✅)**
* Реализация паттерна "Стратегия" (`AbstractProvider`) для поддержки прямого подключения к OpenAI, Anthropic и Gemini API.
* Система Custom API: возможность добавлять любые OpenAI-совместимые сети (OpenRouter, Groq, Ollama, LM Studio) через динамические вкладки.
* Пакетная проверка моделей (Batch Verification): умная очередь пингования моделей с задержкой 1.5 сек (защита от Rate Limit). Автоматическая подмена `max_tokens` на `max_completion_tokens` для o1-совместимых рассуждающих моделей.
* Визуальное управление списком: чекбоксы для показа/скрытия моделей в главном селекторе, сохранение статусов (`✅`/`❌`).
* Система примечаний (Notes): контекстное меню (ПКМ) для добавления личных заметок к моделям ("Хорошо пишет", "Галлюцинирует") и "живой" поиск по списку.
* Умная маршрутизация задач (`AIController`): селектор в статус-баре направляет промпт либо в браузерный мост, либо напрямую в API, автоматически подставляя нужный `Base URL` и ключ. Асинхронное выполнение через `APIWorker`.

**Фаза 19: Встроенный Интеллектуальный Терминал (Integrated Terminal) (Завершено ✅)**
* Интеграция системной командной строки (cmd.exe) прямо под редактором кода через модуль `QProcess`.
* Умная кодировка (CP866 -> UTF-8): полная поддержка русского языка и путей с пробелами и кириллицей в Windows-системах.
* Разделенный UI (Split UX): большое текстовое окно "только-для-чтения" для логов и отдельная строка `QLineEdit` для ввода с поддержкой истории команд (стрелки Вверх/Вниз).
* Авто-активация окружения: при запуске терминал автоматически ищет и незаметно запускает `venv/Scripts/activate.bat`, подготавливая среду к установке пакетов (`pip install`).
* Очистка от ANSI-мусора: регулярные выражения фильтруют цветовые и управляющие коды терминала, сохраняя чистый текст в логах.
* Глобальный шорткат (`Ctrl + ~`) для быстрого скрытия и открытия панели терминала. Перезапуск процесса при смене директории проекта.

## 6. Управление и Горячие клавиши
* `Ctrl + Enter`: Мгновенная отправка задачи из поля ввода.
* `Enter`: Перенос строки в поле ввода.
* `Ctrl + S`: Ручное быстрое сохранение открытого файла (с авто-созданием бэкапа в локальной Машине Времени).
* `Ctrl + ~` (или `Ctrl + \``): Показать/Скрыть нижнюю панель терминала.
* `Стрелки Вверх/Вниз (в терминале)`: Навигация по истории введенных команд.
* `Ctrl + Колесико мыши`: Изменение размера шрифта (поле ввода, логи чата, редактор кода).
* `Клик по кнопке 📦 Git`: Открывает интерфейс просмотра статуса, управления коммитами и облачной синхронизации.
* `Клик по ссылке [Показать текст] / [Показать сырой промпт]`: Просмотр скрытого системного промпта.
* `Drag & Drop файлов`: Перетаскивание файлов из дерева в текстовое поле создает теги упоминаний (`@[core/main.py]`).
* `ПКМ по тегу @файл`: Прикрепляет/открепляет исходный код файла к запросу (цвет меняется на зеленый).
* `ПКМ по файлу в дереве проекта -> 🕒 Машина Времени`: Открывает интерфейс отката старых версий файла из папки `.vibe_backups/`.
* `ПКМ по файлу в окне Git -> 🕒 Посмотреть Git-историю файла`: Открывает окно просмотра коммитов, сравнения Diff и отката к старой версии из облака.
* `ПКМ по файлу в дереве проекта -> 🚫 Добавить в .gitignore`: Автоматически прописывает путь в игнор-лист.
* `Shift/Ctrl + Клик в дереве проекта`: Мульти-выделение файлов для пакетных операций.
* `Левая кнопка мыши (ЛКМ) по иконке ⠿ на плашке в браузере:` Плавное перетаскивание статусной панели по экрану.
* `ЛКМ по имени в плашке браузера:` Вызов диалогового окна для смены человекочитаемого имени вкладки.
* `ПКМ по модели в окне API:` Добавление текстового примечания к модели, копирование текста ошибки или принудительная перепроверка.
* `ПКМ по названию вкладки провайдера (в окне API):` Смена человекочитаемого имени кастомного провайдера (например, с "custom_1" на "OpenRouter").
