# 📖 Руководство пользователя VibeCoder v1.22 (Pro IDE Edition)

Добро пожаловать в **VibeCoder** — вашу персональную среду для профессионального "вайбкодинга". Эта программа работает как умный мост между вашим локальным проектом и веб-версией нейросети Gemini 3.1 Pro (а также внешними API). Она защищает ваш код, обходит ограничения браузеров на длину текста, следит за контекстом и автоматизирует рутину.

---

## 1. Режимы работы: «Чат» против «Кодинга» (Управление контекстом)

В новой версии VibeCoder внедрена двухкнопочная система маршрутизации запросов. Это решает проблему "перегрузки" ИИ лишней информацией, когда вы просто хотите задать вопрос, и наоборот — гарантирует точность, когда ИИ должен переписать пол-проекта.

### 🔘 Кнопка [ 💬 Спросить ] (Горячая клавиша: `Enter`)

**Цель:** Легкий вопрос, теория, объяснение синтаксиса или просьба придумать название функции.

* **Контекст (что отправляется):** Только текст вашего запроса и легкий системный промпт ("Отвечай в Markdown, будь полезным").
* **RAG (База знаний):** ОТКЛЮЧЕН. Мы не тратим время на сканирование проекта.
* **Стейт проекта (структура папок):** НЕ отправляется.
* **Результат:** ИИ отвечает как обычный чат-бот. Программа просто красиво выводит его ответ в окно логов с подсветкой синтаксиса. Если вы руками прикрепили файл (тег `@[main.py]`), ИИ его прочитает, но не будет пытаться прислать JSON для его патчинга.

### 🔘 Кнопка [ ⚡ Кодить ] (Горячая клавиша: `Ctrl + Enter`)

**Цель:** Прямое внесение изменений в файлы проекта, исправление багов, написание новых модулей.

* **Контекст (что отправляется):** Включается «тяжелая артиллерия». ИИ получает жесткий системный промпт Оркестратора, заставляющий его отвечать строго в JSON-формате с блоками `search` и `replace`.
* **RAG (База знаний):** ВКЛЮЧЕН. Система ищет по векторной базе куски кода, семантически похожие на вашу задачу, и незаметно отправляет их ИИ.
* **Стейт проекта:** Отправляется полное дерево файлов и папок проекта.
* **Результат:** ИИ выдает машиночитаемый JSON. Программа перехватывает его, находит нужные строки в ваших файлах и вызывает окно Smart Diff (где зеленым подсвечен новый код, а красным — удаляемый). Вам остается только нажать "✅ Утвердить".

## 2. Архитектура передачи: Browser (Web Gemini) vs API

VibeCoder умеет работать как через интерфейс браузера (бесплатно используя подписку Gemini Advanced), так и через платные API (OpenAI, Anthropic, Groq). Механика передачи данных кардинально отличается.

### Режим API (Прямое подключение)

* **Характер:** Stateless (без сохранения состояния). API ничего не помнит о предыдущих запросах.
* **Механика:** При нажатии "⚡ Кодить" программа собирает гигантский текстовый монолит: системные правила + дерево проекта + RAG-контекст + прикрепленные файлы кода + история переписки. Весь этот мега-промпт отправляется POST-запросом.
* **Плюсы/Минусы:** Работает быстро и предсказуемо, но каждый запрос съедает огромное количество токенов и может стоить дорого.

### Режим Browser (Web Gemini) — Виртуальная файловая система (VFS)

* **Характер:** Stateful. Чат помнит предыдущие сообщения, поэтому мы не гоняем одни и те же правила каждый раз.
* **Проблема:** Если вставить 50 000 символов кода и RAG-индекса прямо в текстовое поле Gemini, вкладка Chrome намертво зависнет, пытаясь отрендерить этот текст и проверить в нем орфографию.
* **Решение (VFS):** VibeCoder вообще не вставляет код в текстовое поле. Вместо этого он на лету запаковывает текстовые данные в виртуальные файлы (`.txt`, `.py`) и "роняет" их в интерфейс Gemini. Парсинг файлов ложится на сервера Google, а ваш браузер работает молниеносно.

## 3. СЕКРЕТ РАЗРАБОТКИ: Как реализована передача инфы файлами (Double-Tap Injection)

*Этот раздел — техническое руководство для применения механизма VFS в других проектах.*

Чтобы обмануть интерфейс и передать гигабайты текста мгновенно, минуя зависание DOM, процесс разбит на 3 этапа:

**Этап 1: Python-backend (Упаковка)**
Вместо того чтобы склеивать строки, `ai_controller.py` переводит текст в кодировку Base64.

```python
b64_rules = base64.b64encode(system_rules.encode('utf-8')).decode('utf-8')
image_payload.append({"mime": "text/plain", "data": b64_rules, "name": "vibe_instructions.txt"})

```

В этот `image_payload` пакуется всё: правила проекта, RAG-выдача, код открытых файлов. В само текстовое поле отправляется лишь 1 строка: *"Посмотри прикрепленные файлы и сделай задачу"*.

**Этап 2: Транспорт (Service Worker)**
Flask-сервер отдает этот JSON-пакет, а расширение Chrome (`background.js`) забирает его через `fetch`. Background-скрипт нужен для обхода CORS-политик Google.

**Этап 3: Браузерная инъекция (Double-Tap в `content.js`)**
Base64 раскодируется обратно в байты, превращается в `Blob`, а затем в нативный JS-объект `File`. Дальше применяется "Двойной бронебойный удар":

1. **Поиск нативного инпута:** Скрипт ищет на странице скрытый `<input type="file">`. Если находит — напрямую перезаписывает его свойство `files` через объект `DataTransfer`. Браузер думает, что пользователь выбрал файлы в проводнике.
2. **Хак с ClipboardEvent (резервный план):** Если Google спрятал инпут, скрипт имитирует нажатие `Ctrl+V` (Вставка). Он создает `ClipboardEvent`, жестко переопределяет свойство `clipboardData` (используя `Object.defineProperty`, так как Chrome блокирует программную вставку файлов по умолчанию) и диспатчит событие прямо в поле ввода Gemini.

Итог: Огромный контекст улетает на сервера Google за миллисекунды в виде аккуратных скрепок с файлами.

## 4. Общение с ИИ, Лимиты и Эстафета

* **Прикрепление файлов:** Введите в чате `@[имя_файла]`, либо перетащите файл из дерева проекта мышкой. Также можно нажать ПКМ по файлу в дереве и выбрать опцию мульти-отправки.
* **Авто-лечение JSON:** Если Gemini сгенерирует сломанный ответ (например, забудет экранировать кавычку в коде), программа не выдаст непонятную ошибку. Она **сама** в фоновом режиме отправит ИИ требование исправить синтаксис, и покажет вам уже готовое окно ревью.
* **Перехват лимитов Pro:** Если у вас закончилась квота на модель Gemini Advanced, расширение это заметит. Программа встанет на паузу и предложит два варианта:
1. Продолжить на Flash (с риском ухудшения качества логики).
2. Нажать **🔄 Эстафета**.


* **🔄 Кнопка "Эстафета":** ИИ сам проанализирует текущий статус проекта, составит бриф (Транзитный пакет) и сохранит его в буфер обмена. Вам останется открыть новую вкладку браузера (с другой учетной записью или когда лимит обновится) и вставить этот бриф. Новый ИИ мгновенно поймет архитектуру и продолжит работу с того же места.

## 5. RAG: Векторная база знаний

Программа обладает "семантическим зрением". Она автоматически режет весь ваш код на мелкие чанки и сохраняет в векторную базу данных ChromaDB.

* **Поллинг:** Программа сканирует файлы каждые 10 секунд. Если вы изменили файл в другой IDE (VS Code или Notepad++), VibeCoder сам заметит это и обновит индекс файла.
* **Как это работает для вас:** Вы пишете `⚡ Кодить: "добавь логирование в сетевые запросы"`. Вы не прикрепляете ни одного файла. RAG сам находит файлы `api_worker.py` и `bridge.py`, пакует их в Base64 и отдает ИИ.

## 6. Машина Времени и Управление версиями (Smart Git)

* **Локальные Бэкапы:** Перед тем как ИИ (или вы через `Ctrl+S`) перезапишет файл, программа делает его слепок. Кликнув ПКМ по файлу -> `🕒 Машина Времени`, вы откроете окно с историей и сможете визуально сравнить текущую версию с любой старой (HTML Diff) и откатиться в 1 клик.
* **Smart Git:** Кнопка `📦 Git` на нижней панели открывает встроенный клиент.
* **AI Commits:** Программа сама анализирует изменения (через `git diff`) и генерирует осмысленные описания коммитов на русском языке.
* **Surgical Pull:** Возможность выборочно скачивать файлы с GitHub, не ломая локальные наработки в других частях проекта.
* **Дерево файлов с чекбоксами:** Удобный выбор файлов для добавления в Staging Area.

## 7. Шпаргалка Горячих Клавиш и Элементов UI

* `Enter` — Спросить (чат).
* `Ctrl + Enter` — Кодить (выполнение задачи с патчингом кода).
* `Shift + Enter` — Перенос строки в поле ввода.
* `Ctrl + S` — Ручное сохранение файла в редакторе (с созданием бэкапа).
* `Ctrl + ~` — Показать/Скрыть нижнюю панель терминала (со встроенным обходом ошибок Traceback).
* `Стрелки Вверх/Вниз (в терминале)` — Навигация по истории команд.
* `Ctrl + Колесико мыши` — Независимое изменение размера шрифта в разных блоках UI.
* `ПКМ по файлу в дереве -> 🚫 Добавить в .gitignore` — Прописать путь в игнор-лист на лету.
* `Shift/Ctrl + Клик в дереве` — Мульти-выделение файлов.
* `Клик по имени проекта (над деревом)` — Открыть Менеджер Проектов (Portable Mode).
