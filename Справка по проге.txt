# 📖 Руководство пользователя VibeCoder v1.28 (Pro IDE Edition)

Добро пожаловать в **VibeCoder** — вашу персональную среду для профессионального "вайбкодинга". Эта программа работает как умный мост между вашим локальным проектом и веб-версией нейросети Gemini 3.1 Pro (а также внешними API). Она защищает ваш код, обходит ограничения браузеров на длину текста, следит за контекстом, автоматизирует рутину и обладает автономными агентными функциями.

---

## 1. Режимы работы: «Чат» против «Кодинга» (Управление контекстом)

В VibeCoder внедрена двухкнопочная система маршрутизации запросов, которая позволяет точно контролировать объем передаваемых данных и экономить токены.

### 🔘 Кнопка [ 💬 Спросить ] (Горячая клавиша: `Enter`)
**Цель:** Легкий вопрос, теория, объяснение синтаксиса или просто общение.
* **Контекст:** Отправляется только текст вашего запроса и легкая системная инструкция («Отвечай как помощник, используй Markdown»).
* **RAG (База знаний):** ПОЛНОСТЬЮ ОТКЛЮЧЕН.
* **Стейт проекта:** Структура папок не отправляется.
* **Результат:** ИИ отвечает как обычный чат-бот. Если вы прикрепили файл тегом `@[main.py]`, ИИ его прочитает, но не будет пытаться прислать JSON для его патчинга. Текст выводится в лог с подсветкой синтаксиса.

### 🔘 Кнопка [ ⚡ Кодить ] (Горячая клавиша: `Ctrl + Enter`)
**Цель:** Прямое изменение файлов проекта, внедрение новых функций, исправление багов.
* **Контекст:** Включается «тяжелая артиллерия». ИИ получает жесткий системный промпт Оркестратора, требующий ответа строго в формате JSON.
* **RAG (База знаний):** ВКЛЮЧЕН автоматически. Система находит в проекте файлы, семантически связанные с вашей задачей, и добавляет их в контекст.
* **Стейт проекта:** Отправляется динамически отфильтрованная структура папок и файлов (без мусора вроде бэкапов, благодаря чтению `.gitignore`).
* **Результат:** ИИ выдает машиночитаемый JSON. Программа перехватывает его и вызывает окно **Smart Diff** (сравнение кода), где вы можете в один клик утвердить изменения.

## 2. Архитектура передачи: Browser (Web Gemini) vs API

VibeCoder умеет работать как через интерфейс браузера (бесплатно используя подписку Gemini Advanced), так и через платные API (OpenAI, Anthropic, Groq, локальные LLM). Механика передачи данных кардинально отличается.

### Режим API (Прямое подключение)
* **Характер:** Stateless (без сохранения состояния). API ничего не помнит о предыдущих запросах.
* **Механика:** При нажатии "⚡ Кодить" программа собирает гигантский текстовый монолит: системные правила + дерево проекта + RAG-контекст + прикрепленные файлы кода + история переписки. Весь этот мега-промпт отправляется POST-запросом.
* **Плюсы/Минусы:** Работает быстро и предсказуемо, но каждый запрос съедает огромное количество токенов и может стоить дорого.

### Режим Browser (Web Gemini) — Декаплинг контекста и защита от зависаний
* **Характер:** Stateful. Чат помнит предыдущие сообщения, поэтому мы не гоняем одни и те же правила каждый раз.
* **Проблема "Слепоты ИИ" и Зависаний:** Ранее весь контекст передавался через Виртуальную файловую систему (VFS), но парсер Gemini часто игнорировал прикрепленный код. С другой стороны, если просто вставить 50 000 символов прямо в текстовое поле Gemini, вкладка Chrome намертво зависнет.
* **Решение (Гибридная доставка v1.28):** * **VFS (Base64-вложения):** Теперь используется ТОЛЬКО для тяжелой базы знаний (`rag_context.txt`) и картинок.
    * **Текст промпта:** Самые важные инструкции, дерево проекта и **исходный код запрошенных файлов** теперь вставляются напрямую в текстовое поле, гарантируя 100% считывание моделью.

## 3. СЕКРЕТ РАЗРАБОТКИ: Visual Handshake и Чанкованная вставка (v1.28)

Это уникальное техническое решение позволяет VibeCoder обходить ограничения безопасности браузеров, предотвращать зависания вкладок и обеспечивать гарантированную доставку контекста.

**Этап 1: Интеллектуальный сбор контекста (Python)**
Сборщик (`ContextBuilder`) на лету читает ваши файлы `.gitignore` и `.vibeignore`, динамически вырезая из структуры проекта мусорные папки (например, архивы и бэкапы). Исходный код запрошенных файлов склеивается с промптом в гигантский текстовый блок.

**Этап 2: Visual Handshake (JavaScript)**
Расширение вбрасывает VFS-файлы (RAG и картинки) в интерфейс Gemini. Скрипт не просто ждет таймер, а интеллектуально мониторит DOM-дерево: он дожидается отрисовки UI-чипов файлов и полного исчезновения спиннеров загрузки (Google CDN), гарантируя, что файлы приняты сервером.

**Этап 3: Бронебойная чанкованная вставка (Hybrid Chunked Paste)**
Чтобы вставить десятки килобайт исходного кода без зависания вкладки (краша React DOM), скрипт применяет гибридный метод:
1. Сначала эмулируется нативное событие `ClipboardEvent 'paste'`.
2. Если защита Google блокирует вставку, скрипт нарезает текст на "чанки" по 3000 символов и вливает их через `execCommand('insertText')` с микропаузами, принудительно удерживая фокус курсора.

## 4. Общение с ИИ, MCP-инструменты и Отладка

* **Прикрепление файлов:** Введите в чате `@[имя_файла]`, либо перетащите файл из дерева проекта мышкой. Также можно нажать ПКМ по файлу в дереве и выбрать опцию мульти-отправки.
* **Инструменты MCP (Model Context Protocol):** ИИ может автономно вызывать локальные инструменты в цикле (до 5 шагов). Ему доступны: безопасная песочница Терминала (выполнение команд вроде `pip install`), чтение баз данных SQLite и бронебойный веб-поиск (DuckDuckGo Lite).
* **Авто-лечение JSON:** Если Gemini сгенерирует сломанный ответ (например, забудет экранировать кавычку), программа сама в фоновом режиме отправит требование исправить синтаксис.
* **Бронебойный парсер (Anti-Silent Fail):** Если ИИ решает поумничать и выдает правильный JSON с выдуманными ключами, программа перехватывает его и выводит в лог как «Мысли ИИ», предотвращая молчаливое зависание.
* **Детектор "Дня Сурка" (Anti-Loop):** Если при вызове инструментов ИИ 3 раза подряд совершает одну и ту же ошибку (сверяется по MD5-хэшу), система жестко прерывает его, требуя сменить стратегию.
* **Инспектор агента:** Кликнув ЛКМ/ПКМ по слову "ВЫ" в окне чата, вы можете открыть окно Инспектора (логи хранятся 7 дней) или экспортировать единый отладочный Debug Report.
* **Anti-Thinking Selector:** При инициализации программа автоматически выбирает в меню Gemini "чистую" модель Pro/Advanced, избегая экспериментальных моделей с принудительным "Thinking", которые ломают генерацию JSON.

## 5. Браузерная интеграция и Вкладки

Для работы Browser-режима необходимо расширение Chrome.

* **VibeBadge (Плашка):** На странице Gemini появится перемещаемая плашка статуса (Heartbeat). 
* **Имена вкладок:** Кликните по названию вкладки на плашке прямо в браузере, чтобы переименовать её (например, "Фронтенд", "Бэкенд").
* **Мульти-маршрутизация:** В статус-баре IDE (внизу слева) есть выпадающий список для выбора целевой вкладки браузера. Это позволяет вести параллельные диалоги для разных частей проекта.
* **Перехват лимитов и Эстафета:** Если лимиты Gemini Pro исчерпаны, программа встанет на паузу. Вы можете нажать кнопку "🔄 Эстафета", чтобы программа сама запаковала весь контекст в "мега-промпт", скопировала его в буфер обмена и позволила новому ИИ продолжить работу с того же места.

## 6. RAG, Машина Времени и Управление версиями (Smart Git)

* **Semantic RAG и Auto-Healing:** Программа режет код на чанки и хранит в векторной базе ChromaDB. Бронебойный поллинг сканирует файлы каждые 10 секунд. Если бинарные индексы на диске повредятся, сработает система Auto-Healing, которая бесшовно пересоздаст БД.
* **Биллинг и «Логические сутки»:** Все затраты токенов на RAG-индексацию ведутся в окне Аналитики. Квоты автоматически сбрасываются ровно в 10:00 утра (00:00 PT). При исчерпании лимита API-ключа, система автоматически берет следующий из пула.
* **Тотальная изоляция:** При смене рабочей папки в IDE, внутренний кэш базы данных принудительно сбрасывается (`RagController.cleanup()`), предотвращая утечку контекста старого проекта.
* **Локальные Бэкапы:** Перед каждой перезаписью файла создается слепок. ПКМ по файлу -> `🕒 Машина Времени` открывает окно HTML-сравнения старых версий с возможностью отката в 1 клик.
* **Smart Git:** Кнопка `📦 Git` открывает встроенный клиент. ИИ сам анализирует `git diff` и генерирует описания коммитов (AI Commits). Доступна функция Surgical Pull для выборочного скачивания файлов с облака.

## 7. Шпаргалка Горячих Клавиш и Элементов UI

* `Enter` — Спросить (режим Чата, легкий вопрос).
* `Ctrl + Enter` — Кодить (выполнение задачи с патчингом кода и полным контекстом).
* `Shift + Enter` — Перенос строки в поле ввода.
* `Ctrl + S` — Ручное сохранение файла в редакторе (с созданием локального бэкапа).
* `Ctrl + F` — Локальный плавающий поиск по открытому файлу.
* `Ctrl + ~` — Показать/Скрыть нижнюю панель терминала (со встроенным обходом ошибок Traceback).
* `Стрелки Вверх/Вниз (в терминале)` — Навигация по истории команд.
* `Ctrl + Колесико мыши` — Независимое изменение размера шрифта в дереве, чате и редакторе.
* `Alt + Left / Alt + Right` (или кнопки ◀ / ▶) — Изолированная Отмена (Undo) и Повтор (Redo) для каждой открытой вкладки.
* `Кнопка ⬇ END / ⬆ TOP` — Прыжок в конец или начало открытого файла (меняет функцию при повторном клике).
* `Shift/Ctrl + Клик в дереве` — Мульти-выделение файлов.
* `ПКМ по файлу в дереве -> 🚫 Добавить в .gitignore` — Динамически прописать путь в игнор-лист.
* `ПКМ по модели в окне API` — Добавление текстового примечания к модели.
* `Клик по имени проекта (статус над деревом)` — Открыть Менеджер Проектов.
* `Символ * на вкладке` — Визуальный индикатор наличия несохраненных изменений в файле.
* `ЛКМ/ПКМ по слову "ВЫ" в чате` — Открытие окна Инспектора агента или сохранение Debug Report.
